# Формат данных для базы программ (Москва + СПб)

Этот документ — техническое задание для исследования (ресерча). Цель: собрать
данные по программам **бакалавриата** Москвы и Санкт-Петербурга в едином формате,
который без переделок вставляется в сайт (`index.html`, массив `PROGRAMS`).

> **Профиль абитуриента, под который заточен сайт:** ЕГЭ **математика (профиль) +
> информатика + русский**, направления вокруг ИИ/ML, Data Science, IT и
> «бизнес + IT». Программы вне этого профиля собирать не нужно (если не указано
> иное).

---

## 0. Главные правила сбора

1. **Не выдумывать.** Если данных нет / не нашли — ставить `null` (для чисел и
   объектов) или `""`/`false` по смыслу. Пустое поле лучше неверного.
2. **Каждый проходной балл — с источником и годом.** В поле-примечании или в
   отдельной колонке `source` указывать ссылку на официальную страницу/документ,
   откуда взята цифра, и год приёмной кампании.
3. **Отличать официальную цифру от оценки.** Если проходной балл — это оценка
   (прикидка по прошлым годам, а не опубликованный балл последнего зачисленного),
   ставить флаг `*Approx = true`.
4. **Один объект = одна образовательная программа** (не направление целиком).
   Если внутри направления несколько профилей с разными проходными — это разные
   объекты.
5. **Баллы — по шкале 0–100 за предмет.** Проходной балл — сумма трёх предметов
   (без индивидуальных достижений), как публикует вуз.
6. **Деньги — в рублях за год**, целым числом, без пробелов и символов (`356000`,
   не `«356 000 ₽»`).

---

## 1. Способ сдачи

Удобнее всего — **одна таблица** (Google Sheets / Excel / CSV) с колонками строго
по списку ниже, **одна строка = одна программа**. Я сам сконвертирую её в JS-формат
сайта. JSON-массив объектов тоже подойдёт — поля те же.

Заголовки колонок должны точно совпадать с именами полей из раздела 2.

---

## 2. Поля (колонки)

### Идентификация и расположение
| Поле | Тип | Обяз. | Описание / допустимые значения |
|---|---|---|---|
| `vuz` | строка | да | Краткое название вуза: `ИТМО`, `МФТИ`, `ВШЭ`, `МГУ`, `СПбГУ`… |
| `faculty` | строка | нет | Факультет/институт/мегафакультет. `""` если нет. |
| `name` | строка | да | Название программы как на сайте вуза. |
| `code` | строка | да | Код направления, напр. `01.03.02`, `09.03.01`. |
| `city` | строка | да | Только `Москва` или `Санкт-Петербург`. |
| `campus` | строка | да | Адрес/район основного кампуса, напр. `Кронверкский пр., 49 · центр`. |
| `campusFar` | bool | да | `true`, если кампус заметно удалён (Петергоф, Долгопрудный и т.п.). |
| `url` | строка | **да*** | Ссылка на **официальную страницу программы** с описанием. *Критично — она выводится кликом по названию.* `null` только если страницы реально нет. |

### Требования ЕГЭ
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `egeSubjects` | строка | да | Человекочитаемый набор ЕГЭ, напр. `математика + информатика + русский` или `математика + физика/информатика + русский`. |
| `egeFits` | bool | да | Принимает ли программа комбинацию **мат+инф+рус** (наш профиль). `false`, если информатику нельзя засчитать. |
| `needsObsh` | bool | да | `true`, если обязателен **обществознание** без альтернативы (делает программу недоступной нашему профилю). |
| `egeCheckNeeded` | bool | да | `true`, если набор ЕГЭ неоднозначен и его надо перепроверять на сайте вуза. |
| `minScores` | объект/`null` | да | Минимальные баллы по предметам, если вуз их задаёт **выше общефедеральных**: `{ "math": 80, "inf": 80, "rus": 60 }`. `null`, если особых порогов нет. Указывать только заданные предметы. |

### Места
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `budget` | число/`null` | да | Всего бюджетных мест. `null` = нет данных. |
| `quota` | число/`null` | да | Оценка мест, уходящих по квотам/БВИ (вычитается из бюджета). `null` = нет данных. |
| `realEge` | число/`null` | да | Реально конкурсных по ЕГЭ мест (≈ `budget − quota`). `null` = нет данных. |
| `budgetOnlyBVI` | bool | да | `true`, если бюджет фактически только для олимпиадников/БВИ (общий конкурс по ЕГЭ ≈ 0). |
| `paid` | число/`null` | да | Платных мест. `null` = нет данных. |
| `noPaidSeats` | bool | да | `true`, если платного приёма нет вообще. |

### Проходные баллы
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `prokhBudget` | число/`null` | да | Проходной балл на **бюджет** (сумма 3 ЕГЭ). `null`, если нет/только платно. |
| `prokhBudgetApprox` | bool | да | `true`, если `prokhBudget` — оценка, а не официальная цифра. |
| `prokhBudgetNote` | строка/`null` | да | Примечание вместо балла, напр. `только платный приём`, `только БВИ`. Иначе `null`. |
| `prokhPaid` | число/`null` | да | Проходной балл на **платное**. `null` = нет данных. |
| `prokhPaidApprox` | bool | да | `true`, если `prokhPaid` — оценка. |
| `prokhPaidYear` | число/`null` | да | Год, к которому относится платный проходной, если не 2025 (напр. `2024`). |

### Деньги
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `priceYear` | число/`null` | да | Цена обучения, ₽/год. `null` = нет данных. |
| `stipend` | число/`null` | да | Повышенная стипендия программы, ₽/мес, если есть. Иначе `null`. |
| `creditAvailable` | bool | да | Доступен ли образовательный кредит с господдержкой (есть платные места). |
| `creditPriceUnknown` | bool | да | `true`, если кредит доступен, но цену не нашли. |

### Классификация (можно оставить пустым — заполню я)
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `filters` | массив строк | нет | Подмножество `["ai","ds","business","it"]`. Если не уверены — оставьте пустым, проставлю по описанию программы. |
| `starAIDS` | 1–5 | нет | Субъективная оценка «насколько это про ИИ/Data Science». Можно не заполнять. |
| `starBusiness` | 1–5 | нет | Оценка «насколько это про бизнес». Можно не заполнять. |
| `profileLabel` | строка | нет | Свободная метка профиля. |

### Доступность
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `unavailable` | bool | да | `true`, если программа недоступна нашему профилю (мат+инф+рус). |
| `unavailableReason` | строка/`null` | да | Причина, если `unavailable=true`. Иначе `null`. |

### Источники (для проверки)
| Поле | Тип | Обяз. | Описание |
|---|---|---|---|
| `source` | строка/`null` | желательно | Ссылка на официальный документ/страницу, откуда взяты места и проходные баллы, + год. |

> Поле `id` НЕ заполнять — присвою при импорте.

## 5. Чек-лист для исследователя

- [ ] Только бакалавриат, только Москва и Санкт-Петербург.
- [ ] Только профиль мат+инф+рус (или близкие; неподходящие помечать
      `egeFits=false` / `unavailable=true`, а не выкидывать молча).
- [ ] У каждой программы — рабочая `url` на официальную страницу.
- [ ] Каждый проходной балл — с годом и ссылкой-источником.
- [ ] Оценочные баллы помечены `*Approx=true`.
- [ ] Нет выдуманных чисел: чего нет — то `null`.
- [ ] Цены и стипендии — целым числом рублей.
- [ ] Заголовки колонок совпадают с именами полей.
