# Куда поступать

Личный инструмент для выбора программ на поступление: калькулятор проходимости,
список «куда подаю» с лимитом 5 вузов × 5 направлений, сравнение программ бок о бок.

Статический сайт (`index.html` + `js/`), без сборки. Данные о программах лежат
в `js/data.js` — правь их напрямую, если появляются уточнения по проходным баллам.

## Как открыть локально

```
python3 -m http.server 8000
```

и открыть `http://localhost:8000`.

## Вкладки

- **Каталог** — все программы с фильтрами (направление, город, дальний кампус),
  для каждой считается статус по баллам: проходишь / на грани / не проходишь /
  бюджет только БВИ / недоступно / нет данных.
- **Мой список** — отмеченные программы, сгруппированные по вузу, со счётчиком
  лимита (не больше 5 вузов, не больше 5 направлений в каждом, не больше 25 всего).
- **Сравнение** — таблица выбранных программ бок о бок: проходной, твой балл,
  стоимость, стипендия, кампус, ссылка.
- **Синхронизация** — необязательное подключение к Firebase, чтобы один и тот же
  список открывался с телефона и с компьютера по общему коду доступа.

## Как посчитан балл

Балл ЕГЭ (284 = математика 94 + информатика 93 + русский 97) + бонус за красный
аттестат/золотую медаль, свой для каждого вуза (`js/data.js` →
`individualAchievements`). Сравнивается с `prokhBudget` каждой программы:
разница ≥ +3 — «проходишь», от −3 до +3 — «на грани», меньше −3 — «не проходишь».
Много `prokhBudget` помечены как `prokhBudgetApprox: true` или с примечанием —
это неподтверждённые цифры, их стоит сверить на сайте вуза перед подачей.

## Синхронизация между устройствами (Firebase)

Сайт статический, поэтому для общего списка между телефоном и компьютером нужен
свой бесплатный Firebase-проект:

1. Зайти на https://console.firebase.google.com/ → создать проект.
2. В нём открыть **Build → Firestore Database** → создать базу (production mode).
3. **Build → Authentication → Sign-in method** → включить **Anonymous**.
4. **Project settings → General → Your apps → Add app → Web (`</>`)** — скопировать
   объект конфига (`apiKey`, `authDomain`, `projectId` и т.д.).
5. Вставить его в `js/firebase-config.js` вместо `null`:
   ```js
   window.FIREBASE_CONFIG = { apiKey: "...", authDomain: "...", projectId: "...", ... };
   ```
6. В Firestore **Rules** поставить:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /lists/{docId} {
         allow read, write: if request.auth != null;
       }
     }
   }
   ```
7. Закоммитить и задеплоить. Во вкладке «Синхронизация» ввести любой код доступа
   (это как пароль — по нему вычисляется идентификатор документа в Firestore) и
   нажать «Подключить» на обоих устройствах с одним и тем же кодом.

Без этой настройки список продолжает работать локально — сохраняется в
`localStorage` браузера, ничего никуда не отправляется.
